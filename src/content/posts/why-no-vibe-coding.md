Coding agents can search, edit, test, and iterate at a speed that changes how software gets built. That does not make judgment optional.

**No Vibe Coding** is about agentic development with evidence: field notes, experiments, and practical patterns for building software with coding agents without outsourcing engineering judgment.

The standard is not whether an agent produced convincing code. The useful questions are:

- What evidence supports the change?
- Which assumptions remain unverified?
- Did the tests exercise the behavior that matters?
- Can another person reproduce the result?
- Where should human judgment interrupt the loop?

This site will document the answers—including the failures.
